# DXF_View

DXF ファイル（`.dxf`）をブラウザだけで閲覧できる、依存ゼロの静的ビューア。
`index.html` 1ファイルで完結。クライアント側でテキストDXFを解析・描画するため、サーバ実行課金は発生しない。

> 旧版（Jw_cad `.jww` ビューア）は親フォルダの `files/` にそのまま残してあります。本フォルダ `files_dxf/` は **読み込み形式を DXF に変更した派生版** です。**UI・操作方法は旧版と同一**で、解析エンジンと名称・アイコンのみ差し替えています。

## 構成

| 項目 | 内容 |
|---|---|
| 形態 | 単一 `index.html`（HTML + CSS + JS インライン） |
| 依存 | なし（CDN・npm パッケージ不要） |
| 解析 | クライアント側で ASCII(テキスト) DXF を直接パース |
| 文字コード | UTF-8 / Shift_JIS（`$ACADVER`・`$DWGCODEPAGE` から自動判別） |
| デプロイ | `index.html` を置いて static hosting するだけ（Vercel / Netlify / 社内Web 可） |

## 対応図形

| DXFエンティティ | 状態 |
|---|---|
| `LINE` | ○ |
| `LWPOLYLINE` / `POLYLINE` | ○ ふくらみ(bulge)円弧は折線近似 |
| `CIRCLE` / `ARC` | ○ |
| `ELLIPSE` | ○ 傾き・扁平・部分弧対応 |
| `POINT` | ○ |
| `TEXT` / `ATTRIB` | ○ 高さ・回転・横倍率・揃え（72/73）対応 |
| `MTEXT` | ○ 整形コード除去・揃え（71）対応 |
| `SOLID` / `TRACE` | ○ 塗り描画 |
| `3DFACE` | ○ 輪郭線描画 |
| `INSERT`（ブロック参照） | ○ 倍率・回転・配列(行列)・入れ子を展開 |
| `DIMENSION`（寸法） | ○ 寸法ブロックを展開して描画 |
| `SPLINE` / `LEADER` | △ 点列を折線近似 |
| `HATCH` | × 未対応 |

- **色**：`BYLAYER`（レイヤ色）/ ACI 色番号 / トゥルーカラー(420) / `BYBLOCK` を解決。色番号1〜9は標準色、それ以外は近似生成。
- **線種**：レイヤ/エンティティの線種名から破線パターンを近似（HIDDEN・CENTER・DASHED 等）。
- **レイヤ**：レイヤ名でフラットに管理。表示 ON/OFF をモーダルで切替（`全表示` / `全非表示`）。

## 機能

- ドラッグ＆ドロップ / クリックで `.dxf` を読み込み
- パン（ドラッグ）、原点固定ズーム（ホイール）、全体表示
- **タッチ操作（iPad）**：Apple Pencil＝点の選択専用／指1本＝選択／指2本＝ピンチ拡大＋移動
- **PDF保存**：現在の表示（図面・寸法込み）をA4 1ページのPDFに保存。初期ファイル名は `<元ファイル名>_yymmdd.pdf`
- **赤連続線記入**：交点・端点をサーチし、□表示された点を順に指定して赤い連続線を記入
- **DXF保存**：赤連続線を `SEKISAN` レイヤ（赤色LINE）としてDXFへ追記して保存
- **周長面積**：`SEKISAN` の赤線で閉じた多角形ごとに周長(m)・面積(㎡)を集計し、中心付近へ `MOJI` レイヤの橙色文字で記入。項目名・数量・単位は個別TEXT
- **寸法記入**：①1点目 → ②2点目 → ③引き出し位置 の3ステップ。交点・端点を最優先にスナップ。`X/Y`（水平・垂直）/ `斜め`
  - DXF座標は実寸（図面単位）のため、寸法値は計測距離をそのまま表示。`縮尺 1/□` で補正可能（既定 1）
- 黒背景 / 白背景の切替（真黒・真白は背景に応じて自動可視化）

## 使い方

1. `index.html` をブラウザで開く
2. 左パネルに `.dxf` をドロップ（同梱の `sample.dxf` で動作確認可）
3. レイヤ・寸法・PDF保存などを操作

### デプロイ（Vercel 例）

```
このフォルダ（files_dxf）直下に index.html を置いて import → Framework Preset: Other → Deploy
```

ビルド不要。`vercel.json` も不要。

## DXF 解析メモ

- **形式**：ASCII(テキスト)DXF を対象。バイナリDXFは未対応（ASCII保存が必要）。
- **トークン化**：`(group code, value)` ペア列に分解し、`HEADER` / `TABLES`（LAYER・LTYPE）/ `BLOCKS` / `ENTITIES` を走査。
- **座標系**：DXF は右手系（Y上向き）。描画時に画面Y下向きへ変換。
- **ブロック展開**：`INSERT` は基準点・倍率・回転・行列を 2D アフィン変換で合成し、入れ子も再帰展開（無限ループ防止に深さ上限あり）。`DIMENSION` は寸法ブロック（`*D…`）を WCS のまま展開。

## 既知の制約

- **HATCH（ハッチング）**：未対応。
- **SPLINE**：制御点／フィット点を折線で近似（厳密なNURBSではない）。
- **トゥルーカラー以外の特殊色**：ACI 10〜249 はHSVで近似生成（AutoCAD標準パレットと厳密一致はしない）。
- **文字フォント**：環境のゴシック体で代替描画（埋め込みフォントは持たない）。
- **非一様倍率の円/円弧**：x方向倍率で近似（楕円は major 軸ベクトルを正確に変換）。
